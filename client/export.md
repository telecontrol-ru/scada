## Экспорт в CSV

Данные из таблиц, сводок, журналов событий и окон наблюдения могут быть экспортированы в файл формата CSV.

Для экспорта данных выберите команду *Экспорт в CSV* из контекстного меню окна или меню *Далее*.

### Параметры экспорта

При экспорте отображается диалог выбора параметров формата CSV:

<dl>

<dt>Кодировка</dt>
<dd>

- Системная кодировка (по умолчанию)
- Unicode (UTF-8)

</dd>

<dt>Разделитель</dt>
<dd>

- Запятая `,` (по умолчанию)
- Точка с запятой `;`
- Двоеточие `:`
- Табуляция
- Пробел
- Произвольный символ

</dd>

<dt>Символ кавычек</dt>
<dd>

- Двойная кавычка `"` (по умолчанию)
- Одинарная кавычка `'`
- Произвольный символ

</dd>
</dl>

Параметры формата сохраняются в профиле пользователя и используются при последующих операциях экспорта.

После завершения экспорта предлагается открыть файл в ассоциированной программе.

## Экспорт в Excel

Таблицы и сводки также поддерживают экспорт данных непосредственно в Microsoft Excel.

## Экспорт конфигурации

Конфигурация объектов системы (группы, ТС, ТИТ) может быть экспортирована в файл CSV для последующего просмотра, редактирования и импорта обратно в систему.

Для экспорта выберите команду *Далее — Экспорт конфигурации*. Файл по умолчанию — `configuration.csv`.

Экспортируемые данные содержат:
* Идентификаторы и имена объектов
* Иерархию (родительские объекты)
* Типы объектов
* Значения всех свойств объектов
* Ссылки между объектами

Формат заголовков: `Имя свойства @ИдентификаторУзла`.

## Импорт конфигурации

Для импорта конфигурации выберите команду *Далее — Импорт конфигурации* и укажите ранее экспортированный CSV-файл.

### Процесс импорта

1. Файл загружается и проверяется на корректность.
2. Выполняется сравнение импортируемой конфигурации с текущей.
3. Отображается отчет об изменениях с тремя категориями:
   * **Создание** — новые объекты, отсутствующие в текущей конфигурации
   * **Изменение** — объекты с измененными свойствами, атрибутами или ссылками
   * **Удаление** — объекты, отсутствующие в импортируемом файле
4. Пользователь подтверждает или отклоняет применение изменений.

**Внимание:** Некорректное использование импорта может привести к потере конфигурации. Внимательно проверяйте отчет об изменениях перед подтверждением.
